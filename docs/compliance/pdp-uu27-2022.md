# UU No. 27 Tahun 2022 — Pelindungan Data Pribadi (PDP)

## Status Classification (2026-02-28)

This document maps AWCMS technical controls to UU PDP requirements. Section 3 describes platform capabilities. Section 4 is an operator action checklist.

## 1. Scope

Indonesia's Personal Data Protection Law (UU No. 27/2022) regulates collection, processing, storage, and transfer of personal data. AWCMS deployments handling Indonesian personal data must comply.

## 2. Key Principles & AWCMS Mapping

| UU PDP Principle | Article | AWCMS Control | Status |
|------------------|---------|---------------|--------|
| **Lawful basis** | Art. 20 | Consent mechanism via `analytics_consent` settings | Implemented |
| **Purpose limitation** | Art. 21 | Tenant-scoped data processing, no cross-tenant sharing | Enforced by RLS |
| **Data minimization** | Art. 22 | Configurable data collection per tenant settings | Configurable |
| **Accuracy** | Art. 23 | User profile edit forms for data correction | Implemented |
| **Storage limitation** | Art. 24 | Configurable retention policies for analytics/audit | Operator responsibility |
| **Integrity & confidentiality** | Art. 25 | RLS, pgcrypto encryption, TLS 1.3 | Enforced |
| **Accountability** | Art. 26 | Audit trail for all write operations | Implemented |

## 3. Data Subject Rights (Hak Subjek Data Pribadi)

| Right | UU PDP Article | AWCMS Support |
|-------|---------------|---------------|
| Right to information | Art. 5-7 | Consent banners plus published privacy/terms pages linked from public routes |
| Right to access | Art. 8 | `UserProfile` view, data export via admin tools |
| Right to rectification | Art. 9 | Profile edit forms for all users |
| Right to erasure | Art. 10 | Soft delete (`deleted_at`); hard delete available to admin |
| Right to restrict processing | Art. 11 | Account suspension and role demotion to No Access |
| Right to data portability | Art. 13 | Data export capabilities via admin API |
| Right to object | Art. 14 | Analytics consent toggle per user |

## 4. Incident Reporting

| Requirement | UU PDP Article | Action |
|-------------|---------------|--------|
| Notification within 3×24 hours | Art. 46 | Operator must configure monitoring and incident response |
| Notification to data subjects | Art. 46(2) | Operator must prepare communication templates |
| Notification to authorities | Art. 46(1) | Operator must know reporting channels |

## 5. Deployer Checklist

- [ ] Identify personal data processed by your AWCMS instance
- [ ] Publish and verify accessible privacy-policy and terms pages for the active tenant/public site
- [ ] Enable consent banners on public portals
- [ ] Configure data retention policies for `analytics_events` and `audit_logs`
- [ ] Prepare incident response procedures (3×24 hour notification)
- [ ] Document lawful basis for each data processing activity
- [ ] Appoint a Data Protection Officer if required by data volume
