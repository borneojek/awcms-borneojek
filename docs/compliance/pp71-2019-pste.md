# PP 71/2019 — Penyelenggaraan Sistem dan Transaksi Elektronik (PSTE)

## Status Classification (2026-02-28)

This document maps AWCMS technical controls to PP 71/2019 requirements. Section 3 describes platform capabilities. Section 4 is an operator action checklist.

## 1. Scope

Government Regulation PP 71/2019 governs the operation of electronic systems and transactions in Indonesia. AWCMS deployments serving Indonesian users or organizations must comply.

## 2. PSE Classification

| Category | Description | Registration |
|----------|-------------|--------------|
| **PSE Lingkup Publik** | Government / public service systems | Register with Kominfo |
| **PSE Lingkup Privat** | Private sector electronic systems | Register with Kominfo |

AWCMS tenants must determine their PSE category and register accordingly.

## 3. Key Requirements & AWCMS Mapping

| Requirement | PP 71/2019 | AWCMS Control | Status |
|-------------|-----------|---------------|--------|
| **System reliability** | Art. 12 | Cloudflare CDN, Supabase managed infrastructure | Platform-level |
| **Data processing security** | Art. 14 | RLS, ABAC, pgcrypto encryption | Enforced |
| **Audit trail** | Art. 16 | `audit_logs` table with actor, action, details | Implemented |
| **Data localization (public sector)** | Art. 21 | Supabase project region selection (Jakarta/Singapore) | Operator config |
| **Access for law enforcement** | Art. 22 | Admin data export, audit log retrieval | Operator responsibility |
| **System documentation** | Art. 13 | SYSTEM_MODEL.md, AGENTS.md, docs/* | Maintained |
| **Incident handling** | Art. 28 | Operator must configure monitoring and response | Operator responsibility |

## 4. Security Governance (Art. 24-27)

| Control | AWCMS Implementation |
|---------|---------------------|
| Access control | ABAC with 10-level hierarchy |
| Authentication | Supabase Auth + optional 2FA |
| Authorization | RLS policies + `has_permission()` |
| Data integrity | Soft delete, foreign key constraints |
| Encryption at rest | Supabase managed + pgcrypto for admin profiles |
| Encryption in transit | TLS 1.3 via Cloudflare |
| Monitoring | Audit logs, platform logs |

## 5. Deployer Checklist

- [ ] Determine PSE category (Publik / Privat)
- [ ] Register with Kominfo via OSS (Online Single Submission)
- [ ] Select appropriate Supabase project region for data localization
- [ ] Configure system monitoring and uptime alerting
- [ ] Prepare incident response procedures
- [ ] Document electronic system architecture (reference SYSTEM_MODEL.md)
- [ ] Ensure terms of service and privacy policy are accessible to users
- [ ] Configure data retention and backup policies
